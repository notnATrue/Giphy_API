Test task for Node.js Developers

The task is to create an intermediate API for images search service through Giphy API. The user can search for image by keywords ( phrases ), mark favorite images and view search history by matching ( phrase ) oneToMany ( favorite images ). As a database you can use any relational database used SQL. The application should work in three states: testing, development, production and covered by end-to-end tests.

Follow these required steps:

  • Create a repository on your GitHub

  • Add to collaborators HELSIS666 and creams9tuna

  • Analyze magic variables and put them into environment variable files:
    .env - for production
    test.env - for testing
    development.env - for development

  • Keep calm and more commits are good :)

  • Write a little documentation on project launches

Testing:

End-to-end application testing includes coverage of all controllers and application methods in two states:

  • Failed
  • Successful

Do not forget that the method may have several cases of error

User flow

Any user in the system must receive his session identifier. Without this identifier, the user cannot perform any actions with the rest of the functionality.

Search flow

Images search is done through the Giphy API. With an empty request, the application should return popular images. Each images in the response from the Giphy API must be mapped to the records of the user’s favorite images.

Favorites flow

The user should be able to mark photos as favorites. Selected images must be unique by an identifier issued by the Giphy API. The user should see these marks in two places:

  • Image Search
  • User history

History flow

User search history must be unique by keyword and include:

  • Keyword
  • Favorites images
  • Timestamp
